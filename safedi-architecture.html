SafeDI Macro Reference

Macro Purpose Example
@Instantiable Opts a type into the SafeDI dependency tree @Instantiable class FavoritesStore { }
@Instantiable(isRoot: true) Marks the entry point of the dependency tree @Instantiable(isRoot: true) class AppDependencies { }
@Instantiable(fulfilledByType:) Maps a concrete type to fulfill a protocol @Instantiable(fulfilledByType: "MovieServiceProtocol")
@Instantiated This type CREATES the dependency @Instantiated let service: MovieServiceProtocol
@Received This type RECEIVES the dependency from a parent @Received private let service: MovieServiceProtocol
@Forwarded Passes runtime values through the tree @Forwarded let authToken: AuthToken

How Data Flows

Step What Happens
1 AppDependencies() is created at app launch
2 SafeDI sees @Instantiated movieService → creates MovieService
3 SafeDI sees @Instantiated favoritesStore → creates FavoritesStore
4 FavoritesStore has @Received service → SafeDI injects movieService
5 Same for MovieListViewModel - gets the same movieService instance
6 Views access dependencies via dependencies.favoritesStore etc.

Before vs After

Before (Manual DI)

// Each class creates its own service
class FavoritesStore {
    init(service: MovieServiceProtocol
         = MovieService()) {  // Hidden default
        self.service = service
    }
}

class MovieListViewModel {
    init(service: MovieServiceProtocol
         = MovieService()) {  // Another instance!
        self.service = service
    }
}

After (SafeDI)

// Explicit dependency declaration
@Instantiable
class FavoritesStore {
    @Received let service: MovieServiceProtocol
    // No default - must be provided
}

@Instantiable
class MovieListViewModel {
    @Received let service: MovieServiceProtocol
    // Same instance shared
}

Legend

Root - Entry point with isRoot: true
Service - Shared service layer
ViewModel/Store - Business logic layer
@Instantiated This type CREATES the dependency (parent owns it)
@Received This type RECEIVES the dependency (child uses it)
